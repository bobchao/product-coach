# 焦點構面深挖（dimension-agnostic）

適用於任何構面——預設八構面或使用者自定義構面都用同一套結構。
一次一問，跟著使用者的回答走，這裡的問句是彈藥不是腳本。

## 結構：As-is → To-be → Actions → Timeframe

### As-is（現在真實的樣子）

目標：讓使用者說出有證據的現狀，不是自我印象。

- 「在這個面向，你現在實際上是怎麼做的？說一個最近兩週的例子。」
- 「這個分數如果要說服別人，你會拿什麼事件當證據？」
- 「你的主管／團隊會怎麼描述你在這件事上的表現？跟你自己的說法一樣嗎？」
- 定稿前可以建議：「要不要先跟 3–5 位常合作的同事要一點具體回饋，
  我們再把 As-is 定下來？」（可跨 session，回饋收齊再定稿。）教他怎麼要：
  請對方講「具體情境、看到的行為、造成的影響」，不要只收形容詞。

### To-be（想成為的樣子）

目標：具體到「別人觀察得到」的程度。

- 「半年後如果這個面向明顯變好了，別人會看到你做什麼不一樣的事？」
- 「有沒有一個你觀察過的人，在這件事上是你想要的樣子？他做了什麼你沒做的？」
- 「是什麼讓你現在想改這個？它跟你目前的工作目標有什麼關係？」
- To-be 太大時：「這是半年可以到的，還是三年的方向？我們先切半年的版本。」

### Actions（接下來的 2–3 個小步驟）

目標：小到下週就能開始、具體到可以打勾。**行動由使用者提出**；
他真的卡住，才明說你要給方向，並提 2–3 個選項讓他選。

- 「朝那個方向，下週你能做的最小的一步是什麼？」
- 使用者想不出行動形式時，可以攤開類型讓他挑：讀一本書、上一門課、
  找一位 mentor、做一個小實驗、主動跟人要 feedback。
- 檢驗每個 action：什麼時候做？在哪個場合？需要誰配合？怎麼知道做了？
- 行動太大時：「這個聽起來要一個月。它的第一個 30 分鐘是什麼？」
- 行動超過三個時：「只能留兩個，你留哪兩個？」
- 依賴他人時：「如果對方沒空，你自己能推進的版本是什麼？」

### Timeframe（時間框架與成功訊號）

- 「這一輪我們看多久？」（預設一個月，見 follow-up.md）
- 「到 checkpoint 那天，看到什麼就算有進展？」——訊號要可觀察，
  接受「做了三次」這種行為訊號，不強求成效指標。
- 習慣型行動（想養成一個固定做法）約需兩個月才穩定——checkpoint
  看的是頻率訊號（這個月做了幾次），不是「已養成」。
- 「中間想被提醒嗎？還是 checkpoint 見？」（順勢介紹週級 micro check-in）

## 收尾規則

計畫成形後，請**使用者**用自己的話完整複述一遍 As-is / To-be / Actions /
Timeframe——複述不出來的部分就是還沒想清楚的部分，回去補。複述完成後
寫入 journey（儲存方式見 SKILL.md 的「Journey 儲存」）。

例外：使用者明確要收尾（「先到這，幫我記下來」）時，**不要用複述擋住
寫入**——journey 先寫（未複述、未確認的事項標為待確認），複述留到
下次回訪開場再補。寧可留一筆待確認的紀錄，不要因為堅持流程而什麼
都沒留下。
